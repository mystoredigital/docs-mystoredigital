Tu Cerebro Digital: de cero a sistema en 1 semana

Quick-start del curso lead magnet. Sistema PARA + Wiki con IA construido sobre Obsidian. Plantilla descargable de GitHub, 7 lecciones cortas, ~85 minutos en total.

Fecha · 2026-05-26 Sistema · Obsidian + Templater + Claude/Cursor

Qué es esto

Una guía web de referencia para los inscritos al curso "Tu Cerebro Digital" publicado en Skool. Sirve para dos cosas: si oíste hablar del curso y quieres entender en 5 minutos qué es, lee esta página completa. Si ya estás inscrito y necesitas un comando, un link o un troubleshooting rápido, esta es tu referencia siempre a la mano.

El curso entrega una plantilla de Obsidian publicada en GitHub bajo licencia MIT — la misma que usamos internamente para organizar empresas, proyectos, lecturas, daily notes y agentes de IA. La descargas, la usas, la modificas, la compartes.

Repositorio público. github.com/mystoredigital/obsidian-vault-template — 76 carpetas, 10 plantillas Templater, sistema Wiki LLM listo para IA.

Qué vas a construir

El vault tiene dos áreas separadas con propósitos distintos:

1. MI CEREBRO DIGITAL/ — Sistema PARA

Inspirado en PARA de Tiago Forte pero adaptado. Carpetas numeradas (el número da orden, el emoji da identidad visual):

2. Wiki/ — Wiki LLM separada

Sistema de conocimiento sintetizado por un LLM (Claude, Cursor, ChatGPT). Tres capas:

  • raw/ — fuentes inmutables (PDFs, transcripciones, recortes). Solo lectura.
  • wiki/ — páginas generadas por el LLM en sources/, entities/, concepts/, analyses/.
  • CLAUDE.md — instrucciones para el LLM con cuatro operaciones: PROCESA, CONSULTA, LINT, EXPLORA.
El diferencial. Antes de los LLMs, mantener una wiki personal era trabajo de doctorado. Ahora un LLM la construye por ti, citando siempre la fuente original en raw/. Tú quedas como editor, no escritor.

Paso a paso (7 lecciones, ~85 min)

01 · Bienvenida + descarga la plantilla — 5 min

Clonas el repo y verificas la estructura. Sin código, sin configuración todavía.

Acción:

git clone https://github.com/mystoredigital/obsidian-vault-template.git mi-cerebro-digital

02 · Filosofía PARA — 12 min

Entiendes por qué organizar por accionabilidad (no por tema) es lo que hace que el sistema sobreviva a las 2 semanas. La pregunta mágica: "¿Tengo que hacer algo específico con esto, en una fecha concreta?"

Acción: identifica en tu vida ahora mismo 1 proyecto con deadline, 1 activo permanente y 1 pieza de conocimiento que quieres consultar después.

03 · Instala Obsidian y configura tu vault — 15 min

Descarga Obsidian, abre el vault, instala los 4 plugins esenciales y configura Templater + Daily notes apuntando a las rutas correctas.

Plugins esenciales: Templater, Dataview, Obsidian Full Calendar, Advanced Tables.

Hotkeys clave: Cmd+T (insertar plantilla), Cmd+Shift+D (daily note de hoy).

04 · El flujo diario — 12 min

El ciclo de vida de una nota: captura → procesa → usa → archiva. La regla maestra: cuando dudes, captura en 001 - Captura y procesa después. Una hora a la semana procesando inbox = sistema vivo.

Acción: captura 3 cosas durante el día. Mañana, decide a qué carpeta va cada una.

05 · Las 10 plantillas — 15 min

Conoces cuándo usar cada plantilla. Templater aplica la correcta automáticamente según la carpeta donde crees la nota.

Acción: crea tu primer proyecto real desde plantilla PROYECTO en 100 - Proyectos.

06 · Wiki LLM con IA — 18 min

La lección estrella. Aprendes la arquitectura de tres capas, las cuatro operaciones (PROCESA, CONSULTA, LINT, EXPLORA) y procesas tu primera fuente real.

Acción: copia un artículo o transcripción a Wiki/raw/, dale al LLM la orden PROCESA Wiki/raw/<archivo> según Wiki/CLAUDE.md y revisa las páginas generadas en Wiki/wiki/.

07 · Próximo paso: comunidad — 8 min

Cierras el curso con tu setup funcionando. Conoces los siguientes pasos: casos avanzados, workshops mensuales, integraciones, agentes IA encima del vault.

Acción: postea tu setup final en la comunidad de Skool.

Comandos útiles (copy-paste)

Clonar la plantilla

git clone https://github.com/mystoredigital/obsidian-vault-template.git mi-cerebro-digital
cd mi-cerebro-digital
find . -name ".gitkeep" -delete   # opcional: limpia los placeholders

Configuración de Templater (Settings → Templater)

  • Template folder location: MI CEREBRO DIGITAL/901 - 🧱 PLANTILLAS
  • Trigger Templater on new file creation: ON
  • Enable Folder Templates: ON

Folder Templates a configurar:

001 - 📥 CAPTURA      → plantilla CAPTURA
100 - ✅ PROYECTOS    → plantilla PROYECTO
150 - 🏆 ACTIVOS      → plantilla ACTIVO
Libros                → plantilla LIBRO
900 - 📆 DIARIO        → plantilla_DIARIO_intersticial

Configuración de Daily Notes (Settings → Daily notes)

  • New file location: MI CEREBRO DIGITAL/900 - 📆 DIARIO
  • Template file location: MI CEREBRO DIGITAL/901 - 🧱 PLANTILLAS/plantilla_DIARIO_intersticial.md
  • Date format: YYYY-MM-DD

Procesar una fuente con LLM en la Wiki

Una vez tienes una fuente cruda en Wiki/raw/<tema>.md, abre tu LLM (Claude Code, Cursor, ChatGPT con archivos) sobre la carpeta del vault y dale:

PROCESA Wiki/raw/habit-stacking.md según las instrucciones de Wiki/CLAUDE.md

El LLM genera:

  • Wiki/wiki/sources/habit-stacking.md — resumen estructurado de la fuente
  • Wiki/wiki/entities/<autor>.md — página de la persona/organización
  • Wiki/wiki/concepts/habit-stacking.md — concepto desarrollado
  • Wiki-links automáticos entre todas las páginas

FAQ rápido

¿Por qué no Notion?

Notion es excelente para wikis colaborativos. Pero su data vive en sus servidores, no es portable, y los LLMs trabajan mejor con markdown plano. Este sistema es local-first: tu vault es tuyo, en tu disco, en archivos de texto que cualquier herramienta lee.

¿Funciona en Windows?

Sí. Obsidian es multiplataforma. La única diferencia es Ctrl en vez de Cmd para hotkeys, y la terminal es Git Bash o PowerShell.

¿Necesito saber código para usar la Wiki LLM?

No. Solo necesitas un LLM con acceso a archivos. Claude Code es lo más cómodo, pero Cursor o ChatGPT con archivos también funcionan. Los comandos definidos en Wiki/CLAUDE.md los das en español o inglés natural.

¿Cómo sé si algo es Proyecto o Activo?

Pregunta: "¿Puedo imaginar el momento exacto en que digo 'ya terminé'?" Si sí → proyecto. Si no → activo. Ejemplos: "mudarme" es proyecto, "mi casa" es activo. "Lanzar el curso" es proyecto, "mi canal de YouTube" es activo.

Las plantillas se ven con <% tp.file.title %> literal

Estás usando el plugin nativo Templates en vez de Templater. Las variables <% %> solo las procesa Templater. Instala Templater desde Community plugins y configúralo apuntando a la carpeta de plantillas.

Tengo 500 capturas viejas sin procesar

No las proceses todas. El 80% no vale tu tiempo. Filtro recomendado: borra todo lo de más de 6 meses (si no lo procesaste, no lo vas a usar). De lo restante, aplica la regla de 30 segundos: si en medio minuto no decides dónde va, archívala o bórrala.

Importante. El sistema no es el cerebro. Es el andamiaje. La fricción de capturar tiene que ser cero; la decisión de organizar viene después, en un ritual semanal de una hora. Si pasas más de 30 segundos pensando dónde va una nota nueva, el sistema ya falló.
